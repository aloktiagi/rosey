What it does

One brain your family actually shares.

Most family logistics live in someone's head, a sticky note on the fridge, or one spouse's phone. Rosey is the third place — a small, addressable memory that takes text from any of you and answers from the same source.

Reminders that don't get lost

Text "remind us about the dishwasher repair tuesday at 2." Rosey schedules it, fires it on time, and chases until someone acknowledges.

Shared facts, addressable

Vet numbers, babysitter rates, mom's pill schedule, the wifi password, the cat's meds. Whoever asks gets the same answer — no "did you write that down somewhere?"

Knows who's who

A roster file maps people to phones. "@Alex, the trash" reaches them on whichever channel they're on. No more cc'ing the family group.

Works on the channels you use

Telegram and WhatsApp today. Alexa and SMS in progress. Same Rosey, same memory — pick the channel that suits each person, whether that's a parent on Telegram, a teen on WhatsApp, or grandma on the kitchen speaker.

Group chat aware

Add Rosey to your family WhatsApp group. She replies in-thread when addressed and stays quiet when she shouldn't butt in.

Yours, not ours

Self-hosted on your Fly.io account. Your memory file lives in your storage. We can't see it. There is no SaaS to discontinue.

How it works

One process. Three doors. One memory.

Every channel hands off to the same agent loop. The agent reads a roster, writes to a shared memory file, and schedules anything time-shaped onto a persistent job queue. Replies fan back out to whichever channel makes sense.

Under the hood

Boring choices in service of reliability.

Reminders are load-bearing for a family. Group chat ergonomics are load-bearing for adoption. The architecture optimizes for both, and not much else.

01 Channels are independent

Each channel is a thin webhook adapter that converts to a canonical identifier (tg:, wa:, alexa:) and hands off. WhatsApp going down doesn't take Telegram with it — including the reminder fan-out.

02 Roster maps people, not devices

One markdown file lists each person's channel identifiers. @Alex in a reminder body resolves to every channel they're on, in parallel. New phone? Edit one line.

03 Reminders survive restarts

APScheduler with a SQLite jobstore on a Fly volume. Each reminder is three jobs: fire, escalate, miss. The container can crash mid-schedule and reconcile picks up where it left off.

04 Group chat without the noise

A fuzzy gate runs before the agent in groups: only addressed messages get a reply. Rosey doesn't interrupt; she answers when summoned.

05 WhatsApp groups via Baileys

Meta's Cloud API can't reach groups without "Official Business Account" status — gated behind well-known brand verification. A Node sidecar runs the MultiDevice protocol directly so families can add Rosey to their group chat today.

Run it yourself

Your house, your stack.

Rosey is a single Python repo, one Dockerfile, one Fly app. About an hour end-to-end if you've shipped a webhook before. Telegram is the easiest starting channel — WhatsApp and Alexa are bolt-ons.

Costs are two line items: ~$5/mo for the Fly.io VM (or $0 if you run her on a Raspberry Pi or home server), plus $5–15/mo of Anthropic API usage for a typical family. Roughly $10–20/mo all in. Current API rates.
